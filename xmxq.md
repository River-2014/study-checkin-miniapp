## 三、命名规范与编码约定

- **文件、目录名**：小写，多个单词用连字符 `-` 连接，如 `home.js`
- **变量/函数**：驼峰命名法 `camelCase`
- **常量**：全大写蛇形 `MAX_STREAK`
- **函数职责单一**：一个函数不超过30行（渲染类除外）
- **注释**：每个自定义函数上方用中文注释说明用途
- **模块化**：工具函数抽离到 `utils/` 目录；每个页面逻辑写在 `Page({})` 内，不污染全局
- **禁止`var`**：统一使用 `const`/`let`
- **避免魔法数字**：重要常量提取到文件顶部

## 五、核心业务规则

### 5.1 打卡规则
- 用户点击任务卡片完成打卡，立即获得该任务积分
- 再次点击可取消打卡（撤回积分）
- 每日至少完成1个任务即视为“今日已打卡”
- 连续打卡天数计算：
  - 今日完成首个任务时，若昨天有任意打卡，连续+1；否则连续=1
  - 取消今日最后一个任务时，若昨天有打卡，连续-1；否则连续=0
- 跨天自动判断：每次进入首页或`onShow`时校验昨天是否有打卡，若前天和昨天均未打卡，连续归零

### 5.2 积分
- 不可为负，最低为0
- 兑换奖励时扣除对应积分，需校验余额充足

### 5.3 徽章系统
使用条件判断自动颁发，徽章不可重复获取。徽章列表如下（针对学习习惯定制）：

| 徽章Key | 名称 | 条件 |
|--------|------|------|
| streak_3 | 连续3天 | 连续打卡≥3天 |
| streak_7 | 一周坚持 | 连续打卡≥7天 |
| streak_14 | 两周达人 | 连续打卡≥14天 |
| streak_30 | 月度冠军 | 连续打卡≥30天 |
| total_50 | 50次打卡 | 累计打卡次数≥50 |
| total_100 | 100次打卡 | 累计打卡次数≥100 |
| stars_100 | 百星富翁 | 累计获得星星≥100 |
| stars_500 | 五星上将 | 累计获得星星≥500 |
| master_math | 数学小达人 | 完成数学任务累计≥30次 |
| master_chinese | 语文小学霸 | 完成语文任务累计≥30次 |
| master_english | 英语小能手 | 完成英语任务累计≥30次 |

### 5.4 鼓励语库（打卡成功随机一句）
“太棒了！离梦想更近了🌟”、“坚持就是胜利💪”、“你是最棒的！🏆”、“今天的努力，明天的收获🌱”、“爸爸妈妈为你骄傲❤️”、“又向前迈了一步📈”、“学习使我快乐😊”、“加油，小升初必胜🔥”

## 八、页面详细设计

### 8.1 首页 `pages/home/home`

**WXML结构**：
1. 头部渐变容器（问候语+星星积分）
2. 连续打卡卡片（火焰图标+天数）
3. 任务进度条
4. 任务列表（`wx:for`渲染，每个任务绑定`data-id`，`bindtap`切换状态）
5. 成就徽章横向滚动区（`scroll-view scroll-x`）

**JS核心方法**：
- `onLoad` -> 加载数据、更新streak、渲染
- `toggleTask(e)` -> 获取id，调用`storage.js`处理数据，重渲染，播放动画
- `checkBadges()` -> 遍历徽章条件，发放新徽章并toast
- `playCelebrate(pts)` -> 随机鼓励语toast + 撒花节点动态添加

### 8.2 日历页 `pages/calendar/calendar`
- 月历生成算法：给定年、月，计算第一天星期几，总天数，前后填充
- 日期格子`bindtap`弹出`wx.showModal`显示当日完成的任务列表

### 8.3 商城 `pages/rewards/rewards`
- 奖励卡片`bindtap`触发`redeemReward`，校验积分，扣除，刷新

### 8.4 我的 `pages/mine/mine`
- 统计卡片
- “添加自定义任务”按钮 -> 弹出底部弹窗，包含：
  - 任务名称input
  - 积分input（数字键盘）
  - emoji选择器（20个常用学习图标横向排列，点击选中高亮）
  - 确认/取消按钮
- “恢复默认任务”按钮（重置tasks为默认）
- “重置全部数据”按钮（双重确认）
