### 技术要点

#### CI/CD 流程设计
1. **代码质量检查** (lint-and-validate)
   - 爬虫代码语法检查
   - 云函数语法检查
   - 配置文件格式校验
   - 敏感信息泄露扫描
   - 大文件检查

2. **自动化测试** (test)
   - 运行单元测试
   - 爬虫功能验证
   - 云函数测试

3. **构建** (build)
   - 安装依赖
   - 代码优化（压缩、混淆）
   - 生成版本信息
   - 上传构建产物

4. **部署云函数** (deploy-cloud-functions)
   - 安装 CloudBase CLI
   - 登录 CloudBase
   - 遍历部署所有云函数
   - 验证部署

5. **部署小程序** (deploy-miniprogram)
   - 下载构建产物
   - 安装 miniprogram-ci
   - 上传小程序代码

6. **部署后验证** (post-deploy-check)
   - 云函数健康检查
   - 冒烟测试

7. **通知** (notify)
   - 发送部署成功/失败通知
   - 支持 Slack、企业微信、钉钉

8. **自动回滚** (rollback)
   - 部署失败时自动回滚
   - 发送回滚通知

#### 关键技术
- **GitHub Actions** - CI/CD 平台
- **miniprogram-ci** - 微信小程序上传工具
- **@cloudbase/cli** - CloudBase CLI 工具
- **多环境管理** - dev/test/prod 环境隔离
- **自动回滚** - 部署失败自动恢复
- **缓存优化** - 加速依赖安装

./deploy/scripts/upload-miniprogram.sh test 1.0.0 "测试部署"

### 技术决策

1. **选择 GitHub Actions 作为 CI/CD 平台**
   - 理由：与 GitHub 仓库集成紧密，免费额度足够使用
   - 替代方案：Jenkins、GitLab CI、CircleCI

2. **使用 miniprogram-ci 上传小程序**
   - 理由：官方推荐工具，支持自动化上传
   - 替代方案：微信开发者工具 CLI（需要 macOS/Windows 环境）

3. **使用 @cloudbase/cli 部署云函数**
   - 理由：CloudBase 官方 CLI，支持批量部署
   - 替代方案：在微信开发者工具中手动上传

4. **多环境管理策略**
   - dev 环境：开发者本地调试
   - test 环境：预发布验证
   - prod 环境：正式对外发布

### 遇到的挑战

1. **微信小程序上传限制**
   - 问题：微信小程序上传需要使用官方工具，且需要私钥文件
   - 解决：使用 miniprogram-ci + GitHub Secrets 管理私钥

2. **云函数部署复杂**
   - 问题：有多个云函数需要逐个部署
   - 解决：编写自动化脚本遍历部署所有云函数

3. **多环境配置管理**
   - 问题：不同环境需要不同的配置
   - 解决：创建环境配置文件 + GitHub Environments
