## 二、核心问题详细分析

### P0 - 致命问题（2项）

#### 2.1 CSS 变量形同虚设

**现状**：app.wxss 定义了 12 个 CSS 变量（--theme-orange, --bg-cream, --text-dark 等），但全项目几乎无页面引用这些变量。所有页面均使用硬编码十六进制颜色值。

**影响**：
- 主题切换（如暗色模式）完全不可能实现
- 品牌色变更需要修改 30+ 个文件
- 视觉一致性无法从架构层面保证

**修复方案**：
```
步骤1：扩展 CSS 变量体系至 30+ tokens
步骤2：编写批量替换脚本（sed/node脚本）
步骤3：逐页面验证替换结果
步骤4：删除所有硬编码颜色值
```

#### 2.2 品牌色不统一

**现状**：主品牌色应统一为 `#FF9F43`，但存在以下变体混用：
| 颜色 | 十六进制 | 使用位置 |
|------|----------|----------|
| 品牌色A | #FF9F43 | app.wxss, home, rewards, calendar 等（主色） |
| 品牌色B | #FF6B35 | wrongbook, ai-exam, daily-plan |
| 品牌色C | #F39C12 | home header渐变, rewards header, guardian |

**修复方案**：统一为 `#FF9F43`，深色变体统一为 `#E8831A`，通过 CSS 变量管控。

### P1 - 严重问题（4项）

#### 2.3 TabBar 图标重复和错配

**现状**（app.json tabBar 配置）：
| Tab | 图标文件 | 问题 |
|-----|----------|------|
| 打卡 | checkin.png | 独立 ✅ |
| 日历 | stats.png | 与错题本共用 ❌ |
| 商城 | profile.png | 与"我的"共用 ❌ |
| 错题本 | stats.png | 与日历共用 ❌ |
| 我的 | profile.png | 与商城共用 ❌ |

**修复方案**：
- 重新设计 5 个独立 TabBar 图标
- 推荐使用 SVG 绘制（81x81px）
- 图标语义：打卡✓/日历📅/商城🎁/错题📝/我的👤

#### 2.4 组件复用率极低

**现状**：
- TAB 切换：detail, family, admin, leaderboard 4+ 页面各自实现
- 弹窗覆盖层：mine(2个), mistakes, wrongbook 各自定义
- 卡片样式：全局 .card 存在但 wrongbook, weekly-report 未使用
- 学科标签：.tag-chinese/.tag-math 等在 home.wxss 和 mistakes.wxss 重复定义

**修复方案**：
- 提取 `tab-switch` 自定义组件
- 提取 `modal-overlay` 自定义组件
- 提取 `subject-tag` 自定义组件
- 强制所有页面使用全局 `.card` 类

#### 2.5 页面背景色不统一

**现状**：
| 页面 | 背景色 | 差异 |
|------|--------|------|
| 大部分页面 | #FFF8E7 | 标准 |
| weekly-report | #F5F6FA | 偏灰蓝 ❌ |
| wrongbook(分包) | #FFF8F0 | 偏暖 ❌ |
| detail(家长) | #FFF8F0 | 偏暖 ❌ |
| login | linear-gradient | 渐变（可接受） |
| password | #FFFFFF | 纯白（可接受） |

**修复方案**：统一使用 `var(--bg-cream)` 即 `#FFF8E7`，特殊页面（login/password）保留独立设计。

#### 2.6 首页信息过载

**现状**：home.wxml 单页包含 15+ 个区块：
1. 骨架屏
2. 头部问候
3. 连续打卡火焰卡片
4. 守护卡操作区
5. 契约进度
6. "一键开始" Hero 卡片
7. 全部完成卡片
8. 进度条
9. 今日推荐练习
10. 任务列表
11. 排行榜入口
12. 番茄钟入口
13. 学习周报入口
14. PK赛入口
15. AI 出题入口
16. 题库练习入口
17. 模拟考试入口
18. 错题本入口
19. 成就徽章
20. 学习看板（图表）
21. AI 学习建议
22. 家长中心入口
23. 分享鼓励卡

**影响**：11-12岁用户认知负荷过重，核心操作（打卡）被淹没在大量入口中。

**修复方案**：
- 核心区（头部→打卡火焰→任务列表→进度条）：保留
- 常用入口（排行榜/番茄钟/PK/AI出题）：合并为2x2功能网格
- 成就/图表/AI建议：移至二级区域或折叠
- 错题本入口：已有独立TabBar，可移除
- 分享卡：移至"我的"页面

### P2 - 中等问题（3项）

#### 2.7 缺少暗色模式支持

微信小程序已支持 `@media (prefers-color-scheme: dark)` 和 `darkmode` 配置。当前 CSS 变量体系为暗色模式提供了可能，但因变量未被引用，暗色模式完全无法实现。

**建议**：在 CSS 变量统一引用后，在 app.wxss 中添加 `[data-theme="dark"]` 变量覆盖即可。

#### 2.8 可访问性严重缺失

| 问题 | 影响 | 修复建议 |
|------|------|----------|
| 无 aria-label / role | 屏幕阅读器无法识别 | 关键交互元素添加 aria-label |
| emoji 做图标无替代文字 | 视觉障碍用户无法理解 | 添加 aria-hidden + sr-only 文字 |
| 颜色对比度部分不达标 | WCAG AA 4.5:1 未满足 | 检查并提升 #B2BEC3 on #FFF8E7 对比度 |
| 无焦点样式指示 | 键盘导航无视觉反馈 | 添加 :focus-visible 样式 |

#### 2.9 排版系统不规范

**现状**：字号散乱分布在 20rpx ~ 64rpx 之间，缺少系统化的 type scale：
- 页面标题：40-44rpx（不统一）
- 卡片标题：30-36rpx（不统一）
- 正文：28-30rpx（基本一致）
- 辅助文字：22-26rpx（不统一）
- 数据展示：36-64rpx（随意）

**建议建立5级排版规范**：
| 级别 | 字号 | 字重 | 用途 |
|------|------|------|------|
| xs | 22rpx | 400 | 标签、提示 |
| sm | 26rpx | 400 | 辅助文字、描述 |
| base | 28rpx | 400 | 正文内容 |
| lg | 32rpx | 600 | 小标题、卡片标题 |
| xl | 40rpx | 700 | 页面标题 |

### P3 - 建议（2项）

#### 2.10 代码冗余

- subpkg-admin 根路径存在 4 个空占位文件（questionManage, stats, reviewQueue, unauthorized），实际功能在 `pages/admin/` 下
- privacy.wxss 与 agreement.wxss 内容完全相同
- 学科标签样式在 home.wxss 和 mistakes.wxss 中重复定义

#### 2.11 "我的"页面信息架构

15+ 个功能卡片纵向排列，滚动距离过长。建议：
- 分组：学习管理 / 奖励管理 / 家庭管理 / 系统设置
- 折叠：低频操作（题库管理、法律隐私）默认折叠
- 层级：家长设置、身份切换等移至子页面

6. **图表组件复用良好**：chart-bar/chart-line/chart-radar 在多页面复用
