## 2. 产品目标

### 2.1 北极星指标

**WDPM（Weekly Deep Practice Minutes）— 周有效深度练习时长**

定义：用户每周完成的有效深度学习总时长，需同时满足：
- 单次连续学习≥5分钟
- 期间无超过2分钟的闲置
- 完成至少1次练习/打卡动作

选择依据：
- DECR仅衡量"是否打卡"，无法衡量学习深度
- 积分比5.6:1根源在于打卡行为过于轻量
- WDPM与学习效果相关性更高，跨学段可比

### 2.2 v2.0 新增目标

| 编号 | 目标 | 目标值 | 依据 |
|------|------|-------|------|
| G1 | 提升学习深度 | WDPM人均≥90分钟/周 | 数据分析：积分比5.6:1说明行为轻量 |
| G2 | 跨学段留存提升 | 小6→初1流失率≤35% | 数据分析：死亡之谷需主动干预 |
| G3 | 积分经济平衡 | 获取/消耗比降至2.0-2.5:1 | 数据分析：当前5.6:1严重失衡 |
| G4 | 品牌认知重建 | 高中生新品牌排斥率≤20% | 用户研究：当前>80%排斥率 |

## 3. 用户故事

### US-6：小学低年级亲子共学

> 作为小学2年级学生的家长，我想和孩子一起完成语音打卡任务，在陪伴中了解学习状态，同时孩子觉得有趣愿意坚持。

验收标准：
- 语音打卡成功后即时反馈（正确/鼓励），家长端收到通知
- 语音识别失败时提示"换个安静的地方试试"，允许重录（最多3次）

### US-7：小学高年级自主打卡

> 作为5年级学生，我想看到我和同学的打卡进度对比，知道自己的位置并更有动力坚持。

验收标准：
- 4-6年级用户可看同班同学打卡天数排行（仅前10名）
- 完成当日全部任务后显示"今日击败了XX%的同班同学"

### US-8：初中中考冲刺

> 作为9年级学生，我想在首页看到中考倒计时和各科薄弱知识点，有针对性地分配复习时间。

验收标准：
- 9年级首页显示中考倒计时+错题本数据标注薄弱知识点（≤3个/科）
- 点击薄弱知识点跳转至AI出题练习

### US-9：高中深度学习

> 作为11年级理科生，我想用番茄钟模式进行深度学习，完成后自动记录学习时长和科目，不被花哨的游戏化元素打扰。

验收标准：
- 10-12年级默认番茄钟界面（25+5分钟），不展示卡通角色和星星动画
- 完成番茄钟自动归入学科学习时长统计，累计WDPM

### US-10：多孩跨学段家庭管理

> 作为有小学3年级和初中7年级两个孩子的家长，我想在一个账号下管理两个孩子的不同打卡任务，切换时体验完全不同。

验收标准：
- 切换孩子时首页UI、打卡方式、积分体系均切换
- 各孩子积分独立计算，不因学段不同导致不公平

## 5. 用户研究洞察（来自瑞思）

### 各学段核心差异

| 维度 | 小学1-3年级 | 小学4-6年级 | 初中7-9年级 | 高中10-12年级 |
|------|-----------|-----------|-----------|-------------|
| 注意力时长 | 15-20分钟 | 25-35分钟 | 30-45分钟 | 45-90分钟 |
| 核心驱动力 | 家长+游戏化 | 同学比较+成就 | 同伴压力 | 自我驱动 |
| 家长角色 | 共学者 | 监督者 | 观察者 | 后勤保障 |
| 游戏化偏好 | 重度(卡通/动画) | 中度(排行榜/PK) | 轻度(数据成就) | 极简(效率工具) |
| 社交需求 | 亲子为主 | 同学比较 | 同伴打卡 | 学伴(轻社交) |
| 隐私需求 | 无 | 初现 | 强烈 | 极强 |
| 打卡频率 | 高频短时 | 中频中时 | 中频中时 | 低频长时 |

### 关键风险（用户视角）

1. **品牌风险**：高中生对"小升初"品牌排斥率>80%（必须更名）
2. **需求冲突**：低年级重游戏化 vs 高年级去游戏化，需学段差异化UI
3. **隐私合规**：14岁以下需监护人同意，跨学段数据迁移需重新授权
4. **家长角色变化**：从共学者→后勤保障，产品需自适应调整

### 核心竞争态势

- **蓝海确认**：「习惯养成×学习内容×游戏化」三角交叉地带无直接竞品

## 8. 积分经济 v2.0

### 分学段定价模型

| 参数 | 小学 | 初中 | 高中 |
|------|------|------|------|
| 基础单次打卡积分 | 10分 | 15分 | 20分 |
| 连续打卡加成 | +2/天(上限+10) | +3/天(上限+15) | +5/天(上限+25) |
| 深度练习奖励 | +5/次 | +8/次 | +12/次 |
| 日均获取量(预估) | 25-35分 | 35-50分 | 45-70分 |
| 最贵奖励积分 | 500分 | 1200分 | 2000分 |
| 最贵奖励兑换周期 | 14-20天 | 24-34天 | 29-44天 |

### 跨学段积分转换

- 小学→初中：1:1.5（购买力等价）
- 初中→高中：1:1.33（购买力等价）
- 转换前弹窗展示汇率，需用户确认

### 通胀控制机制

| 机制 | 方案 |
|------|------|
| 积分衰减 | 小学3%/周、初中2%/周、高中1.5%/周 |
| 衰减启动阈值 | 小学200分、初中500分、高中800分 |
| 保护期 | 小学7天、初中5天、高中3天 |
| 积分上限 | 学段最贵奖励×3 |
| 年级重置 | 小学→初中70%，初中→高中60% |

### 数据门

**M1→M2升级条件**：
1. 5-6年级 DECR ≥ 55%
2. 1-4年级 DECR ≥ 45%
3. D7留存 ≥ 20%
4. 积分获取/消耗比 ≤ 2.5:1
5. 最贵奖励兑换周期 ≥ 14天

**M2→M3升级条件**：
1. 初中 DECR ≥ 45%
2. 小6→初1 GTR ≥ 50%
3. D30留存 ≥ 12%
4. 成长档案创建率 ≥ 30%
5. LTV/CAC ≥ 3:1

## 9. 关键流程

### 9.1 年级设置流程（新用户）
用户打开小程序 → 授权登录 → 选择角色(学生/家长) → [学生]选择年级(1-12) → 自动判定学段 → 展示学科列表 → 确认 → 生成默认任务 → 进入学段首页

### 9.2 年级切换流程（老用户升学）
设置>年级管理 → 修改年级 → 判断是否跨学段 → [同学段]确认→更新 → [跨学段]学段过渡引导→差异说明+成长里程碑 → 14岁以下需监护人授权 → 数据迁移 → 生成新任务 → 进入新首页

### 9.3 多孩跨学段管理流程
我的>孩子管理 → 孩子列表(头像+年级+学段) → 点击切换 → 首页整体切换 → 数据完全独立 → 推送按学段差异化
