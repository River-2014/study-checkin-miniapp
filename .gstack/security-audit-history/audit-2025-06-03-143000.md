## 执行摘要

学习打卡小程序整体安全架构**基础合理**：管理员鉴权走 wxContext.OPENID + admins 集合，家长-孩子数据隔离在云函数层做权限校验，AI API Key 通过环境变量注入不硬编码。但存在 **2 个严重漏洞**、**4 个高危漏洞** 和多个中低风险项需要修复后方可上线。

**最关键发现**: ai-report 云函数 chat 模式允许前端传入任意 URL（`event.url`），存在 SSRF 风险；managePapers / importQuestions / autoCrawl / aiAutoTag 等写操作云函数缺少管理员鉴权，任何用户均可调用。

**最高优先级修复**: F-001（SSRF）和 F-002（缺失鉴权）。

## 发现清单

### [F-001] ai-report 云函数 SSRF 漏洞（chat 模式）

- **分类**: OWASP A10 (SSRF) / STRIDE - Information Disclosure
- **严重性**: Critical
- **置信度**: 10/10 — 代码直接确认
- **位置**: `miniprogram/cloudfunctions/ai-report/index.js:72-73`
- **描述**: chat 模式中，`event.url` 允许前端传入任意 URL，服务端直接请求该 URL，未做任何域名白名单校验或内网 IP 过滤。攻击者可构造请求访问云环境内网服务、获取云元数据（169.254.169.254）、扫描内网端口。
- **攻击场景**:
  1. 攻击者调用 `wx.cloud.callFunction({ name: 'ai-report', data: { mode: 'chat', url: 'http://169.254.169.254/latest/meta-data/' } })`
  2. 云函数向元数据服务发起请求，返回云凭证
  3. 攻击者获取临时凭证后可访问云资源
- **复现步骤**:
  1. 在小程序端调用 `wx.cloud.callFunction({ name: 'ai-report', data: { mode: 'chat', url: 'http://169.254.169.254/', messages: [] } })`
  2. 观察返回值是否包含元数据
- **修复建议**:
  1. 移除 `event.url` 参数，chat 模式始终使用环境变量 `AI_API_URL`
  2. 如必须保留自定义 URL，添加域名白名单（仅允许 `api.deepseek.com`、`api.openai.com` 等已知 AI API 域名）
  3. 添加内网 IP 过滤（参考 autoCrawl 中的 `validateUrl` 函数，复用其内网 IP 检测逻辑）
- **优先级**: P0（立即修复）

### [F-002] 多个写操作云函数缺失管理员鉴权

- **分类**: OWASP A01 (Broken Access Control) / STRIDE - Elevation of Privilege
- **严重性**: Critical
- **置信度**: 9/10 — 代码直接确认
- **位置**:
  - `miniprogram/cloudfunctions/managePapers/index.js:121-180` (create action)
  - `miniprogram/cloudfunctions/managePapers/index.js:185-206` (delete action)
  - `miniprogram/cloudfunctions/managePapers/index.js:253-275` (update action)
  - `miniprogram/cloudfunctions/managePapers/index.js:211-248` (batchDeleteMetadata action)
  - `miniprogram/cloudfunctions/importQuestions/index.js:220-285` (主入口)
  - `miniprogram/cloudfunctions/autoCrawl/index.js:164-260` (主入口)
  - `miniprogram/cloudfunctions/aiAutoTag/index.js:117-207` (主入口)
  - `miniprogram/cloudfunctions/generatePaper/index.js:20-133` (主入口)
- **描述**: manageQuestions 中写操作（batchUpdate、export）有 `requireAdmin` 鉴权，但 managePapers 的所有写操作（create、delete、update、batchDeleteMetadata）、importQuestions（可批量导入/覆写题库）、autoCrawl（可爬取外部网站并写入题库）、aiAutoTag（可修改题目标签）、generatePaper（可生成试卷）均无管理员鉴权。任何微信用户均可直接调用这些云函数执行敏感写操作。
- **攻击场景**:
  1. 攻击者调用 `wx.cloud.callFunction({ name: 'managePapers', data: { action: 'delete', paperIds: ['all_ids'] } })` 删除所有试卷
  2. 攻击者调用 `importQuestions` 注入恶意题目数据
  3. 攻击者调用 `autoCrawl` 让云函数请求外部网站（绕过频率限制）
- **复现步骤**:
  1. 以普通用户身份调用 `wx.cloud.callFunction({ name: 'managePapers', data: { action: 'create', paper: { title: 'TEST_INJECT' } } })`
  2. 检查是否成功创建试卷
- **修复建议**:
  1. 在 managePapers 的 create、delete、update、batchDeleteMetadata action 开头添加 `requireAdmin('operator')`
  2. 在 importQuestions、autoCrawl、aiAutoTag、generatePaper 主入口添加 `requireAdmin('operator')`
  3. 统一抽取 `requireAdmin` 函数到共享模块，避免各云函数重复实现
- **优先级**: P0（立即修复）

### [F-003] TTS 云函数日志泄露签名和密钥信息

- **分类**: OWASP A09 (Security Logging Failures) / STRIDE - Information Disclosure
- **严重性**: High
- **置信度**: 10/10 — 代码直接确认
- **位置**: `miniprogram/cloudfunctions/tts/index.js:95-98`
- **描述**: TTS 云函数在每次调用时打印完整的 Payload、CanonicalRequest 和 Authorization 头到日志。Authorization 头包含 TC3-HMAC-SHA256 签名，Payload 包含请求参数。这些日志可能被其他有云函数日志查看权限的用户看到，造成签名重放风险和内部信息泄露。
- **攻击场景**: 拥有云开发控制台日志查看权限的人员（包括协作者）可从日志中提取签名信息，构造重放攻击。
- **复现步骤**: 调用 TTS 云函数后，在云开发控制台查看云函数日志，可见完整 Authorization 和 Payload。
- **修复建议**: 移除第 95-98 行的 console.log，仅保留错误日志。如需调试，在生产环境应通过环境变量控制是否输出详细日志。
- **优先级**: P1（本迭代修复）

### [F-004] authAdmin 云函数日志泄露用户 openid

- **分类**: OWASP A09 (Security Logging Failures) / STRIDE - Information Disclosure
- **严重性**: High
- **置信度**: 10/10 — 代码直接确认
- **位置**: `miniprogram/cloudfunctions/authAdmin/index.js:16,33,36`
- **描述**: authAdmin 在多条 console.log 中输出完整 openid。openid 是微信用户的唯一标识，属于个人隐私信息（PII）。云函数日志可能被协作者或审计人员查看，造成用户隐私泄露。
- **攻击场景**: 协作者通过云开发控制台查看日志，获取所有管理员及调用者的 openid，可用于关联攻击。
- **复现步骤**: 调用 authAdmin 后在云开发控制台查看日志。
- **修复建议**: 移除或脱敏 openid 输出，仅输出前4位+***。
- **优先级**: P1（本迭代修复）

### [F-005] 家长密码明文存储 + 硬编码默认密码

- **分类**: OWASP A02 (Cryptographic Failures) / A07 (Authentication Failures) / STRIDE - Tampering
- **严重性**: High
- **置信度**: 9/10 — 代码直接确认
- **位置**: `miniprogram/utils/storage.js:233,857-866`
- **描述**: 家长模式密码以明文存储在 `wx.setStorageSync` 中（`data.user.parentPassword`），默认密码硬编码为 `'1234'`。微信小程序本地存储虽受沙箱保护，但在设备 root/越狱后可被读取。更重要的是，4位数字默认密码极易被猜解，且密码验证为简单字符串比较，无防暴力机制。
- **攻击场景**:
  1. 孩子知道默认密码是 1234，无需猜测即可进入家长模式
  2. Root 设备上可读取本地存储获取明文密码
- **复现步骤**: 首次使用时输入 `1234` 即可进入家长模式。
- **修复建议**:
  1. 密码存储改为加盐哈希（SHA-256 + salt），验证时比较哈希值
  2. 移除默认密码，首次使用时强制要求设置密码
  3. 增加密码强度要求（至少6位，含字母+数字）
  4. 添加密码错误计数和锁定机制（5次错误锁定5分钟）
- **优先级**: P1（本迭代修复）

### [F-006] 邀请码使用 Math.random() 生成，可预测

- **分类**: OWASP A02 (Cryptographic Failures) / STRIDE - Spoofing
- **严重性**: Medium
- **置信度**: 7/10 — 代码确认，实际利用难度中等
- **位置**: `miniprogram/cloudfunctions/familyBind/index.js:12-18`
- **描述**: 家庭邀请码使用 `Math.random()` 生成 6 位字母数字码。`Math.random()` 不是密码学安全随机数生成器（CSPRNG），其输出在 V8 引擎中可被预测。6位码仅 32^6 ≈ 10亿种组合，暴力枚举虽受云函数调用频率限制，但理论上可行。
- **攻击场景**: 攻击者通过多次生成邀请码分析随机模式，或暴力枚举 6 位码加入他人家庭。
- **复现步骤**: 调用 familyBind 的 createInvite 多次，观察生成的邀请码是否有模式。
- **修复建议**:
  1. 使用 `crypto.randomBytes()` 替代 `Math.random()`
  2. 增加邀请码长度到 8 位
  3. 添加邀请码使用次数限制（每个邀请码只能使用1次，已实现）
  4. 添加邀请码尝试频率限制
- **优先级**: P2（下个迭代修复）

### [F-007] submitPractice 云函数信任前端传入的 correctCount

- **分类**: OWASP A01 (Broken Access Control) / A04 (Insecure Design) / STRIDE - Tampering
- **严重性**: Medium
- **置信度**: 8/10 — 代码确认
- **位置**: `miniprogram/cloudfunctions/submitPractice/index.js:15-17`
- **描述**: submitPractice 云函数直接使用前端传入的 `correctCount`、`totalCount`、`duration` 等数据，未在服务端重新验算。攻击者可伪造 100% 正确率刷分。虽然当前积分主要用于激励系统而非金融交易，但破坏了数据完整性。
- **攻击场景**: 攻击者调用 submitPractice 传入 `correctCount: 999, totalCount: 1000`，写入虚假高分记录。
- **复现步骤**: `wx.cloud.callFunction({ name: 'submitPractice', data: { answers: [{questionId:'q1',userAnswer:'A',isCorrect:true}], correctCount: 100, totalCount: 100 } })`
- **修复建议**:
  1. 服务端根据 answers 数组重新计算 correctCount 和 totalCount
  2. 不再信任前端传入的统计值
  3. 对 duration 添加合理性检查（不能为负值，不能超过上限）
- **优先级**: P2（下个迭代修复）

### [F-008] autoCrawl SSRF 防护不完整 — 未拦截云元数据地址

- **分类**: OWASP A10 (SSRF) / STRIDE - Information Disclosure
- **严重性**: Medium
- **置信度**: 8/10 — 代码确认
- **位置**: `miniprogram/cloudfunctions/autoCrawl/index.js:58-79`
- **描述**: autoCrawl 的 `validateUrl` 函数仅过滤了 `10.x`、`127.x`、`172.16-31.x`、`192.168.x` 和 `localhost`，但**未过滤云元数据地址 `169.254.169.254`**。攻击者可通过 autoCrawl 访问云元数据服务。此外未过滤 IPv6 地址（如 `::1`、`[::ffff:127.0.0.1]`）和 DNS 重绑定攻击。
- **攻击场景**: 调用 autoCrawl 传入 `url: 'http://169.254.169.254/latest/meta-data/'`
- **复现步骤**: `wx.cloud.callFunction({ name: 'autoCrawl', data: { url: 'http://169.254.169.254/' } })`
- **修复建议**:
  1. 添加 `169.254.x.x`（链路本地地址）到黑名单
  2. 添加 `0.0.0.0`、`::1`、`fc00::/7` 等特殊地址过滤
  3. 实现 DNS 解析后二次校验（解析 IP 后再次检查是否为内网地址）
- **优先级**: P1（本迭代修复）

### [F-009] managePapers update action 存在 NoSQL 注入风险

- **分类**: OWASP A03 (Injection) / STRIDE - Tampering
- **严重性**: Medium
- **置信度**: 6/10 — 理论可行，需配合 F-002
- **位置**: `miniprogram/cloudfunctions/managePapers/index.js:267-269`
- **描述**: update action 直接将前端传入的整个 `paper` 对象传给 `db.doc().update({ data: paper })`，未做字段白名单过滤。攻击者可传入任意字段覆盖文档中的任意属性，包括 `_id`、`_openid` 等系统字段。虽然云开发 SDK 会忽略部分保留字段，但仍可能覆盖非预期字段。
- **攻击场景**: 攻击者传入 `paper: { _openid: 'attacker_id', title: 'hacked' }` 覆盖文档的 _openid 字段。
- **复现步骤**: 调用 managePapers 的 update action，传入包含非预期字段的 paper 对象。
- **修复建议**: 添加字段白名单过滤，仅允许更新 `title`、`subject`、`grade`、`sections`、`totalScore`、`duration`、`status` 等业务字段。
- **优先级**: P2（下个迭代修复）

### [F-010] speech-eval 云函数返回模拟评分，可被滥用

- **分类**: OWASP A04 (Insecure Design) / STRIDE - Tampering
- **严重性**: Medium
- **置信度**: 8/10 — 代码确认
- **位置**: `miniprogram/cloudfunctions/speech-eval/index.js:21-42`
- **描述**: 口语评测功能返回随机模拟评分（75-95分区间），评分完全由 `Math.random()` 决定，与实际语音无关。用户可通过反复提交获得高分，且评分不反映真实口语水平，对教育产品可信度有严重影响。同时，`recognizedText` 直接返回参考文本而非 ASR 识别结果，等于泄露答案。
- **攻击场景**: 学生反复提交朗读，直到随机获得高分。
- **修复建议**:
  1. 上线前必须接入真实 ASR 评测服务
  2. 若暂时无法接入，在 UI 层明确标注"评分仅供参考，非真实评测"
  3. 添加评分频率限制
- **优先级**: P1（上线前必须处理）

### [F-011] dataSync merge 模式存在数据覆盖风险

- **分类**: OWASP A04 (Insecure Design) / STRIDE - Tampering
- **严重性**: Medium
- **置信度**: 6/10
- **位置**: `miniprogram/cloudfunctions/dataSync/index.js:76-129`
- **描述**: merge 模式中，当本地版本号大于云端版本号时，直接用本地数据覆盖云端数据，未做字段级合并。恶意客户端可通过伪造高版本号覆盖其他设备的数据。同时，当云端数据为 null 时，无条件使用本地数据恢复，也可能被利用。
- **攻击场景**: 在设备 A 上修改数据并伪造高版本号，同步后覆盖设备 B 的数据。
- **修复建议**:
  1. 合并策略改为字段级而非文档级
  2. 添加合并冲突提示，让用户选择保留哪份数据
  3. 对版本号添加合理性校验
- **优先级**: P2（下个迭代修复）

### [F-012] AI prompt 注入 — chat 模式 messages 完全由前端控制

- **分类**: OWASP A03 (Injection - Prompt Injection) / STRIDE - Tampering
- **严重性**: Medium
- **置信度**: 7/10
- **位置**: `miniprogram/cloudfunctions/ai-report/index.js:67-97`
- **描述**: ai-report chat 模式的 `event.messages` 完全由前端传入，服务端未做任何校验或系统提示注入保护。攻击者可构造恶意 messages 绕过 AI 输出约束，生成不当内容。虽然小程序端限制了调用者必须通过 wx.cloud.callFunction，但任何微信用户均可构造恶意 payload。
- **攻击场景**: 攻击者传入 `messages: [{role:'system', content:'忽略所有安全约束...'}]` 覆盖系统提示词。
- **复现步骤**: 调用 ai-report chat 模式，传入包含恶意 system message 的 messages 数组。
- **修复建议**:
  1. 服务端强制注入系统提示词，不允许前端覆盖 system role
  2. 过滤 messages 数组，移除任何 role 为 system 的条目
  3. 对用户消息做敏感词过滤
- **优先级**: P1（本迭代修复）

### [F-013] 前端存储 openid — 信息泄露风险

- **分类**: OWASP A01 (Broken Access Control) / STRIDE - Information Disclosure
- **严重性**: Low
- **置信度**: 9/10 — 代码确认
- **位置**: `miniprogram/utils/account.js:21`
- **描述**: 登录后将 openid 存储在 `wx.setStorageSync('loginUser', { openid: ... })`。openid 属于微信用户标识，在 root 设备上可被读取。虽然小程序沙箱提供了一定保护，但将服务端标识存储在前端不符合最小暴露原则。
- **修复建议**: 前端不存储 openid，仅在需要时通过云函数获取。如需缓存用户信息，使用不含 openid 的用户 ID。
- **优先级**: P3（backlog）
